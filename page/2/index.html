网络基础TCP/IP

发表于 2019-11-07 | 分类于 HTTP

HTTP协议在应用层上

TCP/IP按照层次分别分为：

  • 应用层：决定了向用户提供应用服务时通信的活动，应用服务一般有（FTP文件传输协议、DNS域名系统、HTTP协议）
  • 传输层：提供处于网络连接中的两台计算机之间的数据传输，传输层有两个性质不同的协议（TCP，传输控制协议和UDP，用户数据报文协议）
  • 网络层：处理网络上流动的数据包，数据包是网络传输的最小单元。该层规定了传输路径，IP网际协议在网络层
  • 数据链路层：用来处理连接网络的硬件部分

初探 Nginx 架构

发表于 2019-10-30 | 分类于 Nginx 
众所周知，Nginx 性能高，而 Nginx 的高性能与其架构是分不开的。那么 Nginx 究竟是怎么样的呢？

Nginx 在启动后，在 unix 系统中会以 ***守护进程*** 的方式在后台运行
后台进程包含一个 master 进程和多个 worker 进程

我们也可以手动地关掉后台模式，让 Nginx 在前台运行
并且通过配置让 Nginx 取消 master 进程，从而可以使 Nginx 以单进程方式运行。

很显然，生产环境下我们肯定不会这么做
所以关闭后台模式，一般是用来调试用的
Nginx 是以多进程的方式来工作的，当然 Nginx 也是支持多线程的方式的
我们主流的方式还是多进程的方式，也是 Nginx 的默认方式。
